# Cycle Anchor 数据处理任务完成报告

## 项目概述
完成了对 mcPHASES 数据集中 `hormones_and_selfreport.csv` 的 cycle anchor 相关数据处理工作，包括数据提取、清洗、周期识别和排卵日标注。

## 已完成任务清单

### 1. 数据提取与初步分析 ✅
- **任务描述**：从原始数据集中提取 cycle anchor 相关字段
- **完成内容**：
  - 从 `hormones_and_selfreport.csv` (5659行 × 22列) 中提取了6个关键字段：
    - `id`: 受试者ID
    - `study_interval`: 研究间隔
    - `day_in_study`: 研究中的天数
    - `phase`: 月经周期阶段（Menstrual, Follicular, Fertility, Luteal）
    - `lh`: 黄体生成素水平
    - `estrogen`: 雌激素水平
  - 生成了 `cycle_anchor.csv` 文件（5659行 × 6列）

### 2. 数据质量检查 ✅
- **任务描述**：分析缺失值情况
- **完成内容**：
  - 统计了各字段的缺失值比例：
    - `id`, `study_interval`, `day_in_study`: 0% 缺失
    - `phase`: 0.02% 缺失（1个值）
    - `lh`: 5.65% 缺失（320个值）
    - `estrogen`: 5.67% 缺失（321个值）

### 3. 周期识别与分组 ✅
- **任务描述**：实现两级分组系统（大组和小组/周期）
- **完成内容**：
  - **大组（Big Group）**：按 `id + study_interval` 分组，共识别出 62 个大组
  - **小组（Small Group/Cycle）**：实现周期识别算法
    - 周期定义：从 "Menstrual" 阶段开始，到下一个 "Menstrual" 阶段之前结束
    - 处理了周期前段（cycle 0）：如果数据开始不是 "Menstrual"，则标记为 cycle 0
    - 共识别出 241 个周期（包括 cycle 0）
    - 其中 192 个为常规周期（cycle >= 1）

### 4. 周期级别的缺失值分析 ✅
- **任务描述**：分析每个周期的数据完整性
- **完成内容**：
  - 统计了每个周期的缺失值情况
  - 发现 94 个周期（39.00%）存在缺失值
  - 常规周期中，82 个周期（42.71%）存在缺失值

### 5. 数据清洗与完整周期提取 ✅
- **任务描述**：提取无缺失值的完整周期数据
- **完成内容**：
  - 筛选出 110 个完整周期（无任何缺失值）
  - 生成了清洗后的数据集 `cycle_anchor_clean.csv`
  - 初始数据量：2819 行，110 个周期
  - 验证：清洗后的数据集无缺失值

### 6. 日期连续性检查与数据清理 ✅
- **任务描述**：检查周期内日期的连续性，移除不连续周期
- **完成内容**：
  - 实现了日期连续性检查算法
  - 发现 2 个不连续周期：
    - `18_2024 Cycle 1`: 缺失第 925-927 天（Follicular → Fertility 过渡期）
    - `18_2024 Cycle 2`: 缺失第 974 天（Follicular 阶段内）
  - 移除了 `18_2024 Cycle 1`
  - 最终数据量：2796 行，109 个完整周期

### 7. LH Surge 检测算法实现 ✅
- **任务描述**：实现基于 LH 水平的排卵日检测算法
- **完成内容**：
  - **基线计算**：
    - 定位月经结束日（最后一个 Menstrual 阶段）
    - 计算月经结束后 1-6 天的 LH 平均值作为基线
    - 使用去噪方法：排除最大值和最小值后计算平均值
    - 处理边界情况：无月经期时使用周期前6天
  
  - **Surge 检测**：
    - 从基线期后第7天开始检测
    - 识别 LH/基线 >= 2.0 的连续天数段
    - 如果存在多个 surge，选择最后一个（最接近排卵）
  
  - **排卵日标注**：
    - **Method 1**: Surge 第一天 + 1 天
    - **Method 2**: Surge 峰值日 + 1 天（如有多个峰值，选择第一个）
  
  - **处理结果**：
    - 成功处理了 109 个周期
    - 为多个周期标注了排卵日

### 8. 数据保存与输出 ✅
- **任务描述**：保存最终处理结果
- **完成内容**：
  - 保存了包含排卵日标注的最终数据集
  - 输出文件：`cycle_anchor_clean.csv`
  - 包含字段：
    - 原始字段：`id`, `study_interval`, `day_in_study`, `phase`, `lh`, `estrogen`
    - 新增字段：`ovulation_day_method1`, `ovulation_day_method2`（0/1 标注）
