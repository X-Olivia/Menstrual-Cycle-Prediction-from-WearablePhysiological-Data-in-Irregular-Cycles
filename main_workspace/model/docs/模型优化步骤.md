# 模型优化步骤（科研级执行顺序）

**核心原则**：先确认信号可学习，再调模型，最后调损失。否则可能在优化一个根本不可学习的任务。

---

## 阶段零：排卵信号可学习性验证（必须先做）

在改任何损失、λ 或结构之前，必须回答：

> **排卵是否能被当前特征区分？**

### 做法：Logistic Regression Probe

- **输入**：当天特征 x_t（与当前模型相同的 14 维，或同一批序列的逐日特征）
- **输出**：ovulation 标签（二值：如每周期概率最大日为 1 其余为 0；或软标签阈值化）
- **任务**：逐日样本（不利用序列），用当日 x_t 预测「该日是否为排卵日」
- **指标**：**AUC**、**PR-AUC**

### 判断规则

**AUC**（probe 用 `class_weight=None`，反映真实信号强度）：

| AUC | 结论 |
|-----|------|
| < 0.55 | 没信号 |
| 0.55–0.65 | 弱信号 |
| 0.65–0.75 | 可学习 |
| > 0.75 | 强信号 |

**PR-AUC**：必须相对 **baseline PR（Test 正类比例）** 解读。看 **PR-AUC / baseline_PR**：

| 倍数 | 含义 |
|------|------|
| < 1.5 | 信号弱 |
| 1.5–3 | 有信号 |
| > 3 | 很强 |

### 实现要点（与审稿意见一致）

- **低置信周期过滤**：仅保留 `max(ovulation_prob_fused) >= 0.6` 的周期，避免用噪声标签训练。
- **class_weight=None**：probe 目的是检测信号是否存在，不人为改变分布。
- **Sanity check**：运行时会打印 `Feature mean |x|`；若 ≈ 0 需警惕填 0 过多或标准化错误。

**若不先做这一步，后面所有优化都可能白费。** 可直接运行：

```bash
cd main_workspace && python -m model.baseline_ovulation_probe
```

脚本用同一份数据与 subject 划分，逐日特征 → Logistic Regression → 输出 AUC、PR-AUC 及结论。若首轮即 AUC &lt; 0.55，说明当前特征下排卵几乎不可分，应优先做阶段一（特征消融、缺失处理）或接受上限，再重跑 probe。

---

## 阶段一：特征与信号来源（紧接阶段零）

**目的**：在确认「单日特征至少有一定可分性」后，回答**排卵信号主要来自哪些特征**，避免在不可学习的特征组合上堆模型。

| 序号 | 步骤 | 说明 | 涉及文件 |
|------|------|------|----------|
| 1.1 | **特征消融（排卵 probe）** | 用同一 LR probe 或极简模型：仅温度、仅温度+HR、仅 HRV、全特征等，分别看 AUC/PR-AUC。必须回答「排卵信号到底来自哪个 feature」。 | `config.py`（FEATURE_COLS）、probe 脚本 |
| 1.2 | **缺失填充策略** | 温度/HRV 缺失时用个体内插值或「缺失指示 + 邻日均值」替代全局填 0，再跑 probe，看指标是否提升。 | `dataset.py` 或上游 `processed_data` 管道 |

**运行**：在 main_workspace 下执行 `python -m model.baseline_ovulation_probe`（默认即阶段一消融）；阶段零全特征 probe 用 `python -m model.baseline_ovulation_probe full`。

**验收**：至少有一组特征（如温度或温度+HR）在 probe 上 AUC > 0.55；若全部 < 0.55，优先改特征/标注或缺失处理，再回到阶段零重测。若某组（如仅HRV）明显优于全特征，可据此做特征强调或缺失改进。

**实验记录（阶段一 1.2）**：对 HRV/温度/HR 做缺失填充（插值或缺失指示），保证 HRV 少被填 0 后重跑消融，**「仅HRV」与「全特征」均未提升、反而下降**。结论：当前数据下该填充策略不利于排卵 probe，暂不采用；主模型与 probe 建议沿用原填 0 或仅做轻量缺失处理。可能原因：插值平滑掉日级变异、缺失指示改变分布或引入额外维度噪声等。后续可接受当前信号上限，改在**序列模型内**强调 HRV（如排卵头仅用 HRV 或对 HRV 分支加权），或从标注/他源特征入手。

## 阶段二：损失与训练协议（在信号可学习前提下）

| 序号 | 步骤 | 说明 | 涉及文件 |
|------|------|------|----------|
| 2.1 | **排卵 BCE 正类加权** | 对 `ovulation_prob_fused > 0` 的步提高权重（如正类权重 = 负类步数/正类步数），缓解极端类别不平衡。**必做。** | `losses.py` 或 `train.py`（per-step weight） |
| 2.2 | **Stage 2 保留排卵** | Stage 2 不丢弃 \(L_{ov}\)：排卵头用较小学习率（如 1e-4）仍优化，或用小 λ（如 0.2）保留 \(L_{ov}\)，避免微调阶段完全丢掉排卵信号。**关键改动。** | `train.py`（`run_stage2`）、`config.py` |
| 2.3 | **早停用 ov 的 BCE，不用 corr** | Stage 1 选 checkpoint 时，若需兼顾排卵，用**排卵头的 BCE loss**（或总损失），不要用 `ovulation_corr` 做早停：corr 对噪声敏感，易误杀好模型。 | `train.py`（`run_stage1`） |
| 2.4 | **提高 Stage 1 的 λ（次优先）** | 仅在阶段零/一显示「信号可学习」后再做。将 `LAMBDA_OV` 从 0.5 提到 1.0–2.0；若信号弱，提 λ 无意义。 | `config.py` |

**不建议**：

- **仅在正类步上算 \(L_{ov}\)**：会让模型学不到「非排卵日长什么样」，负类边界缺失，属监督错误。正确做法是**保留所有样本，对正类加权**。

**验收**：Val/Test 的 `ovulation_corr` 上升、主任务 MAE/acc_1d 不显著变差。

## 阶段三：数据与标签（在阶段二之后）

| 序号 | 步骤 | 说明 | 涉及文件 |
|------|------|------|----------|
| 3.1 | **Focal Loss（推荐）** | 在保留「所有样本参与、正类加权」的前提下，用 Focal Loss 替代普通 BCE，进一步减轻负类主导；若保留软标签可对正类做 focal 式加权。 | `losses.py` |
| 3.2 | **二值化标签（可选）** | 若用 Focal，可将软标签二值化（如每周期概率最大日为 1，其余为 0）再训练，便于解释与对比。 | `dataset.py` 或标签脚本 |

**再次强调**：不要改为「只在正类步上算 loss」，必须保留负类样本并加权。

## 阶段四：评估与监控（贯穿全程）

| 序号 | 步骤 | 说明 | 涉及文件 |
|------|------|------|----------|
| 4.1 | **固定汇报 ovulation_corr** | 每次实验必报 Val/Test 的 ovulation_corr，与 MAE、acc_1d 并列。 | `run_experiment.py`、`eval_metrics.py` |
| 4.2 | **按 horizon 分层，重点 6–20** | 优化目标可侧重 horizon 6–20（黄体期）的 acc_1d；21+ 单独记录。 | `eval_metrics.py`、`run_experiment.py` |
| 4.3 | **可选：排卵日 ±1 天召回/精确率** | 定义「排卵日」为每周期概率最大日，算预测峰在真实 ±1 天内的召回率。 | `eval_metrics.py`（新增函数） |

---

## 阶段五：结构改进（仅在有可学习信号且阶段二～三有收益后）

| 序号 | 步骤 | 说明 | 涉及文件 |
|------|------|------|----------|
| 5.1 | **两阶段建模** | 先判断排卵/已过排卵，再在「已过排卵」步上用距排卵天数或黄体期阶段预测月经。**必须在确认 baseline 有可学习信号后再做**，否则结构成本高、收益不确定。 | 新模块或扩展 `net.py`、`train.py`、`dataset.py` |
| 5.2 | **黄体期长度先验** | 在损失或输出层加入黄体期 12–16 天先验，约束排卵后预测区间。 | `losses.py` 或 `net.py` |

---

## 科研级执行路线（按天）

| 时间 | 动作 |
|------|------|
| **Day 1** | 只做一件事：**logistic regression baseline**（排卵 probe），输出 AUC、PR-AUC。 |
| **Day 2** | 若 **AUC > 0.6** → 信号存在，进入阶段一（特征消融）与阶段二（损失/Stage2）；若 **AUC < 0.55** → 信号太弱，必须先改 feature 或数据，再重跑 probe。 |
| **Day 3+** | 在 probe 与消融有结论后，再考虑 λ 调整、Focal Loss、两阶段模型。 |

---

## 当前最需要避免的误区

- **错误心态**：模型没学到 → 一定是模型设计不够好。  
- **更常见现实**：模型没学到 → **信号本来就不可学习**（或极弱）。  
若直接调损失、λ、架构而不先验证信号，可能连调数日指标几乎不变，且无法判断是「调参不够」还是「任务上限」导致。

---

## Baseline 记录（便于对比）

- **Val**: MAE=6.97, acc_1d=13.8%, ovulation_corr=-0.079  
- **Test**: MAE=7.79, acc_1d=15.0%, ovulation_corr=-0.047  
- **Test by horizon**: 1–5d acc_1d≈22.5%, 21+ acc_1d≈7.5%

**优化目标**：先通过 **probe AUC** 确认可学习性；再让 **ovulation_corr** 由负转正并提高；**horizon 6–20 的 acc_1d** 提升；整体 **acc_1d** 向 ±1 天靠近。
