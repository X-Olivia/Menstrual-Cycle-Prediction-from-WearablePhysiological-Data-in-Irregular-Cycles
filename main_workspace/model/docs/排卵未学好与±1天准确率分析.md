# 为什么模型学不到排卵日？——从计算机科学与医学视角的分析

## 〇、排卵头的任务定义（判断 vs 预测）

排卵头是**判断头/概率分类头**，不是预测头：

- **给定**：x₁, x₂, …, x_t（到今日为止的生理信号）
- **输出**：P(今天是排卵日 | 到今日为止的生理信号)

即**在线判别**：根据当前及历史序列，判断「今天」是排卵日的概率；**不**预测未来哪天排卵。排卵是单点事件，可穿戴信号具有滞后性（体温在排卵后上升；输入中无 LH）。生理上：LH 峰在排卵前（未作为输入）；体温上升在排卵后；HRV 多为相位级。因此在 **wearable-only** 下，**x_t → ovulation_t 的映射本身极弱**，排卵头仅能提供弱归纳偏置；主任务仍为月经预测。下文的「学不到排卵」指的就是这一**判断**任务（ovulation_corr≈0）未学好。

## 一、运行结果摘要与问题定位

### 1.1 当前指标

| 集合 | MAE  | acc_±1d | acc_±2d | acc_±3d | **ovulation_corr** |
| ---- | ---- | -------- | -------- | -------- | ------------------------ |
| Val  | 6.97 | 13.8%    | 24.3%    | 36.1%    | **-0.079**         |
| Test | 7.79 | 15.0%    | 24.9%    | 35.6%    | **-0.047**         |

按 **horizon**（真实「距下次月经天数」）分层：

- **1–5 天**：MAE≈4.3–5.4，acc_1d≈22–23%（相对最好）
- **6–10 天**：MAE≈4.3–4.5，acc_1d≈19–23%
- **11–15 天**：MAE≈5.0–5.1，acc_1d≈12–14%
- **16–20 天**：MAE≈5.1–5.6，acc_1d≈13–14%
- **21+ 天**：MAE≈12.1–13.4，acc_1d≈6.5–7.5%（**最差**）

结论：**排卵判断概率与真实排卵概率几乎无相关（略呈负相关）**，说明辅助任务没有学到有效的「今日是否排卵日」信号；**horizon 越大（越在卵泡期）预测越差**，与「若能先判断排卵再预测会更好」的直觉一致。

### 1.2 设计目标与现状

- 论文/设计：用排卵作为**归纳偏置**，约束潜状态 \(h_t\) 编码「与排卵相关的生理转折」，从而改善主任务（月经预测）。
- 你的目标：**±1 天准确率**；理想流程是「序列 → 判断排卵日 → 再预测则很准」。
- 现状：排卵**判断**头几乎没学到（ovulation_corr≈0 且略负），整体 acc_1d≈15%，远未达到 ±1 天。

## 二、计算机科学视角：为什么排卵判断任务学不好

### 2.1 标签极度稀疏与类别不平衡（主因）

数据统计（当前 `cycle_clean_2` + `full` 对齐后）：

- 周期内总时间步：**4825**
- `ovulation_prob_fused` **非零** 的时间步：**360**（约 **7.5%**）
- 即约 **92.5%** 的步目标为 0，仅 **7.5%** 为 (0, 1] 的软标签。

每个周期内：

- 排卵标注非零的天数：**平均 2.1 天，最多 4 天**
- 周期平均长度约 **28 天**，即每周期仅 2–4 天有正类信号。

因此：

- 辅助损失 \(L_{ov}\) 在 **每一步** 都计算 BCE（mask 内），但 **绝大多数步的目标是 0**。
- 优化器为降低 BCE，会倾向于「处处预测接近 0」；少数 360 步的正标签难以在梯度上占主导。
- 结果：排卵判断头几乎总是输出低概率，与真实「今日是排卵日」概率曲线相关性接近 0 或略负（ovulation_corr 为负）。

这是典型的 **极端类别不平衡**：正类样本过少、且分散在不同周期，表征 \(h_t\) 难以被驱动去编码「当前是否在排卵日附近」。注意：这里讨论的是**判断**（P(今日是排卵日 | 序列)）学不好，不是「预测未来排卵日」。

### 2.2 主任务主导与表征冲突

- 主任务（月经）：**每一步** 都有监督（除截断边界），且目标 `days_until_next_menses` 变化平滑（随 t 单调递减）。
- 辅助任务（排卵）：**稀疏** 且 **峰值型**（仅少数天概率高）。

总损失为 \(L = L_{menses} + \lambda L_{ov}\)（当前 \(\lambda=0.5\)）。由于：

- \(L_{menses}\) 参与步数远多于 \(L_{ov}\) 的有效信号步数；
- 主任务更易下降（回归目标明确），

梯度上 **主任务占优**，\(h_t\) 更倾向于编码「距下次月经」或「周期阶段」的平滑信息，而非「今日是否排卵日」的尖峰。
换言之：**同一表征同时要拟合「平滑的剩余天数」和「单日排卵判断概率峰」，两者对 \(h_t\) 的要求存在冲突**，在数据与损失设计下，主任务胜出。且 wearable-only 下 x_t → 排卵日 的判别信号本就极弱（无 LH、体温滞后、HRV 相位级），进一步加剧判断头学不动的现象。

### 2.3 两阶段训练进一步削弱排卵

- **Stage 1**：联合优化 \(L_{menses} + \lambda L_{ov}\)，选模型依据 **总验证损失** \(val\_l\)。
- **Stage 2**：只优化主任务 \(L_{menses}\)，骨干低学习率或冻结，**排卵头不再更新**。

因此：

- 最终用于推理的模型，其排卵头与骨干在 Stage 2 完全为「主任务」服务；
- 早停与最佳 checkpoint 也是按主任务（或总损失）选的，**没有任何机制在 Stage 2 保留或强化排卵信号**。

结果是：即便 Stage 1 学到一点排卵相关表征，Stage 2 的微调也会让表征进一步偏向主任务，排卵头输出与真实排卵概率的相关性难以维持，甚至变差（与观测到的轻微负相关一致）。

### 2.4 特征维数与缺失：不是「太多」，而是「质量与权重」

- 当前输入为 **14 维**：HRV（rmssd, lf, hf, lf_hf_ratio）、HR（mean, std, min, max）、腕温（wt_*）、夜间温度、静息心率等，均为个体内 z 标准化。
- 维数 14 对 GRU 来说并不算高；问题更在于：
  - **缺失**：很多天 HRV 等被填 0（`FILL_MISSING=0`），若缺失比例高，有效信号以温度、心率为主；
  - **与排卵最相关的生理量**（体温/腕温）混在 14 维中，若缺失多或方差小，模型可能更依赖 HR/HRV，而文献表明 **排卵的穿戴设备信号中体温/腕温最可靠**（见下节）。
- 因此：**不是「特征太多导致判断错」**，而是 **排卵相关特征（尤其是温度）在序列中可能权重不足或噪声大**，再加上 2.1–2.2 的稀疏与主任务主导，排卵**判断**头难以利用到这些特征。任务本身是「判断今日是否排卵日」的在线判别，不是预测未来排卵日。

## 三、医学视角：排卵在可穿戴信号中的可辨识度（为何判断也难）

### 3.1 体温与排卵：事后性、个体差异与漏检

- **基础体温（BBT）/腕部皮肤温度（WST）** 在排卵**后**出现 **双相升高**（约 0.3–0.5°C），并维持至下次月经。
- 医学事实：
  - 仅约 **70%** 的排卵周期会呈现清晰双相 BBT 模式；约 **20%** 有排卵但 BBT 无典型双相。
  - BBT 最低点与 LH 峰关系密切，但 **日测 BBT 对「排卵当日」的判别/确认** 多为 **事后**（升温确认已排卵），且 LH 未作为输入。
- 因此：**判断「今天是不是排卵日」** 时，最直接的事前信号（LH surge）不可用；体温是**排卵后**才升，对「当日」判断是滞后、弱信号。若夜间温度缺失多或个体内变异大，可穿戴温度在日级别上对排卵日的**判断**本身就是 **弱信号、事后性** 的靶子。

### 3.2 黄体期相对稳定：为何「先判排卵再预测」是合理目标

- **黄体期**（排卵后至下次月经）长度相对稳定，多数在 **12–16 天**，变异小于卵泡期。
- 若能在序列中 **识别「已过排卵」**（或排卵日附近），则「距下次月经」≈ 黄体期剩余天数，预测会明显变准；**卵泡期**（horizon 21+）因长度变异大、且无排卵锚点，预测自然更差——与当前按 horizon 分层结果一致。
- 因此：**从医学上，「先判断排卵再预测」是合理且重要的设计目标**；当前模型在 21+ 天 horizon 上 MAE≈12–13、acc_1d≈7%，恰恰说明缺少有效的「排卵锚点」利用。

### 3.3 文献中可穿戴设备与排卵/月经预测

- 可穿戴 + 机器学习在 **月经/生育窗** 上的结果大致为：
  - **生育窗/排卵相关**：准确率约 85–87%（多为窗而非单日）；**相位分类**（月经期 / 排卵期 / 黄体期）可达约 87% 准确率、0.96 AUC。
  - **排卵单日** 的精确估计通常 **未** 作为主要指标报道，多数工作做的是「窗」或「相位」，而非「±1 天排卵日」。
- **月经预测**：有报道 3 天前预测约 83–90% 准确率；**±1 天** 的严格指标在公开文献中较少见，说明从可穿戴到「±1 天」本身是较高目标。
- 结论：**仅凭可穿戴、不引入激素，要稳定学好「今日是否排卵日」的判断本身就具有生理与文献上的难度**（x_t → ovulation_t 映射极弱）；当前设计（稀疏软标签 + 主任务主导）进一步放大了这一困难。

## 四、综合结论与原因归纳

| 层面                | 原因                                                                                                                                         |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **数据/标签** | 排卵标签极度稀疏（约 7.5% 步非零）、每周期仅 2–4 天有正信号，BCE 被大量 0 主导，正类梯度弱。                                                |
| **任务设计**  | 主任务每步监督、目标平滑；排卵为稀疏尖峰。共享\(h_t\) 时主任务占优，表征更拟合「距月经」而非「排卵日」。                                     |
| **训练协议**  | Stage 2 只训主任务且排卵头不更新，早停按主任务/总损失，排卵信号在后期被进一步边缘化。                                                        |
| **生理/信号** | 排卵是判断而非预测；无 LH 输入、体温在排卵后上升、HRV 为相位级 → wearable-only 下 x_t→排卵日 判别极弱；体温事后性、约 20% 周期无典型双相。 |
| **特征**      | 维数不是主要问题；温度等排卵相关特征若缺失多或未突出，会进一步削弱排卵的可学习性。                                                           |

因此：**不是「特征选得太多导致排卵判断错」**，而是 **排卵（判断）标签稀疏、主任务主导、两阶段未保护排卵、以及生理上 wearable-only 对「今日是否排卵日」的判别信号极弱** 共同导致模型学不好排卵判断；进而无法利用「排卵锚点」做准黄体期预测，±1 天准确率难以提升。

## 五、改进方向建议（兼顾 CS 与医学）

### 5.1 缓解排卵标签稀疏与不平衡

- **重加权 BCE**：对 `ovulation_prob_fused > 0` 的步提高权重（如正类权重 = 负类样本数/正类样本数），使正类梯度不被淹没。
- **仅在有排卵标注的周期/步上算 \(L_{ov}\)**：若当前已是「仅 mask 内」计算，可检查 mask 是否覆盖了过多「标注为 0」的步；考虑 **仅在有非零排卵概率的周期内** 对排卵头做监督，或对「排卵日附近 ±1 天」赋予更高权重。
- **二值化 + 焦点损失**：将软标签二值化为「排卵日/非排卵日」（例如取概率最大日为 1，其余为 0），并采用 Focal Loss 减轻负类主导（若仍保留软标签，也可对正类做 focal 式加权）。

### 5.2 显式「先排卵、后月经」的建模（对齐你的目标）

- **两阶段推理/训练**：
  - 阶段一：用当前序列（或仅温度/HR 子集）训练「今日是否排卵日」的**判断**（或「是否已过排卵」分类）；
  - 阶段二：在「判断已过排卵」的步上，用「距排卵天数」或「黄体期阶段」作为额外输入或条件，再预测 `days_until_next_menses`。这样显式把「判断排卵」作为中间变量，更符合「先判排卵再预测」的医学逻辑。
- **结构化先验**：在损失或输出层加入「黄体期长度」先验（如 12–16 天的分布），约束「排卵后」的预测落在合理区间；或对「排卵日附近」的预测误差加大惩罚，鼓励模型先学好排卵再细化月经预测。

### 5.3 多任务与训练协议

- **延长或调整 Stage 1**：在 Stage 1 中增大 \(\lambda\)（如 1.0–2.0）或对排卵头使用更大学习率，使排卵在联合训练中占更大比重；早停可考虑同时看 `ovulation_corr`，避免选到「主任务好但排卵完全丢失」的 checkpoint。
- **Stage 2 保留排卵**：Stage 2 不冻结排卵头，或对骨干采用极小学习率同时仍保留 \(L_{ov}\)（小权重），避免微调阶段完全丢弃排卵信号。

### 5.4 特征与输入

- **突出温度相关特征**：若存在 `nightly_temperature`、腕温等，可做单独分支或额外归一化，确保温度在序列中不被 HRV/HR 淹没；或做 **特征选择/消融**，验证「仅温度」「温度+HR」等对排卵相关指标的影响。
- **缺失与填充**：对温度缺失多的天，考虑用插值或「缺失指示+邻日均值」而非全局 0，减少噪声对排卵学习的干扰。**注**：实际在阶段一 1.2 中做了 HRV/温度/HR 的插值或缺失指示后重跑 probe，仅HRV 与全特征均下降，已记录于《模型优化步骤》；当前建议沿用原填 0 或轻量处理。

### 5.5 评估与目标

- **±1 天** 作为主指标时，可 **按 horizon 分层报告**（你已在做），并重点优化 **horizon 6–20**（黄体期为主）的 acc_1d；同时监控 **ovulation_corr**（排卵**判断**概率与真实概率的相关性）或「排卵日附近 ±1 天」的召回/精确率，确保改进确实来自排卵判断的利用而非仅主任务过拟合。
- 若数据中排卵金标准有限，可考虑将「相位分类」（月经/卵泡/排卵/黄体）作为中间评估，与文献中的 87% 等结果对照，再衔接「排卵判断→月经预测」的流程。

## 六、简短总结

- **任务界定**：排卵头是**判断头**——P(今天是排卵日 | x₁..x_t)，在线判别，不预测未来排卵日。wearable-only 下 x_t→排卵日 的映射极弱（无 LH、体温滞后、HRV 相位级）。
- **为什么学不好排卵判断**：主任务梯度主导共享表征，Stage 2 又只优化主任务，加上可穿戴对「今日是否排卵日」的判别信号本身偏弱且事后性，导致 ovulation_corr≈0 或略负。
- **与「特征太多」的关系**：特征维数不是主因；更关键的是排卵相关信号（尤其是温度）的权重与质量，以及上述任务/训练/标签设计。
- **为何对 ±1 天重要**：医学上黄体期相对固定，先**判断**排卵再预测能显著提升准确率；当前 horizon 21+ 的差表现印证了「缺排卵锚点」是瓶颈。
- **建议优先尝试**：排卵判断损失重加权/正类加权、显式「先判断排卵后月经预测」的两阶段建模或结构化先验、Stage 2 保留排卵相关训练、以及强化温度特征与缺失处理；同时用 ovulation_corr 与分层 acc_1d 做系统评估。
