# 排卵 Probe 结果差的原因分析

## 当前现象

- **Cycle-level accuracy = 0/7**，随机基线约 6.3%
- **Per-day AUC ≈ 0.48**（无信号）
- 提示：`Feature mean |x| (sanity): 274749` —— 正常 z 分数应在约 ±3 内，均值绝对值应在 0.5～1 量级

## 原因分析（按优先级）

### 1. 数据质量是主要问题（优先解决）

**现象**：`processed_data/2/full.csv` 中部分行的 **z 分数字段不是真 z 分数**，出现极大/极小值。

示例（节选）：

| 列           | 合理范围 | 实际出现 |
|--------------|----------|----------|
| hr_mean_z    | ≈ ±3     | -8731237 |
| hr_std_z     | ≈ ±3     | -3044386 |
| hr_max_z     | ≈ ±3     | 10000000 |
| hr_min_z     | ≈ ±3     | -11000000 |
| wt_mean_z    | ≈ ±3     | 261168   |
| resting_hr_z | ≈ ±3     | 79771002 |

**来源（已定位并修复）**：

- 在 **`data_process/daily_data_2.ipynb`** 的 STEP 6 中，滚动 z 分数公式为 `z = (x_t - μ_t) / σ_t`，其中 μ_t、σ_t 为过去 K=14 天的均值和标准差。
- 当过去 K 天**方差为 0**（如 resting_hr 多天缺失被填 0、或某特征多天常数）时，原代码用 `σ = eps = 1e-6`，导致 `z = (x - μ) / 1e-6` 爆炸（例：resting_hr 从 0 变为 79 → z ≈ 79e6）。
- **修复**：同一 notebook Cell 11 已改为：若 `σ < eps` 则置 `z = 0`；否则计算 z 后裁剪到 ±5（`Z_CLIP = 5`）。重新跑完 STEP 6–7 并覆盖 `processed_data/2/full.csv` 后，下游 probe 的 z 尺度即正常。

**影响**：

- 线性 probe 被这些异常值主导，学不到“排卵 vs 非排卵”的规律
- 即便用 StandardScaler，分布已被破坏，模型难以利用

**建议**：

1. **上游修复**（推荐）：在生成 `full.csv` 的流程里  
   - 检查 z 分数计算（按被试或按全局？分母是否为 0？）  
   - 对 z 分数做裁剪（如 ±5 或 ±4）或 winsorize，再写入  
2. **Probe 内临时缓解**：已在 `baseline_ovulation_probe` 中对特征做 `clip(-5, 5)`，避免单点爆炸主导；修复数据后可将 `FEATURE_CLIP` 调大或取消。

---

### 2. 测试集周期数很少，指标方差大

- 当前按被试 85/15 划分，**测试集只有 7 个周期**。
- Cycle-level accuracy = 正确周期数 / 7，少 1 个就对就差约 14%，容易得到 0/7。
- 因此“0%”既可能来自模型真差，也可能来自样本量小的高方差。

**建议**：

- 数据修好后，可考虑略提高测试比例或做多折（按被试划分），看多折上 cycle accuracy 的均值和方差。
- 报告时注明“测试周期数 = 7”，避免过度解读单次 0/7。

---

### 3. 是否“数据量太少”？

- 训练约 1160 天、76 个正例；测试 231 天、14 个正例。
- 对**线性** probe 来说，样本量不算极少，但在**特征被异常值污染**的前提下，加数据也难以改善。
- 结论：**先解决特征质量，再判断是否缺数据**。

---

### 4. 是否需要加入“先验/领域知识”？

- **当前阶段**：先不要依赖复杂先验。问题主要是**特征不可信**（z 分数异常），修好数据并重跑 probe 后再看 AUC/cycle accuracy 是否提升。
- **若修好数据后仍很差**，再考虑：
  - **简单先验特征**：例如“距下次月经天数”“周期内相对日”等，帮助模型利用“排卵多发生在黄体前”等规律。
  - **约束/规则**：例如排卵日约束在某个周期阶段、与体温/HRV 的文献规律结合等。
- 若修好数据后 probe 已有明显信号（如 AUC > 0.6、cycle accuracy 明显高于随机），可优先继续从特征与模型结构优化，再按需加先验。

---

## 建议行动顺序

1. **修 z 分数**：检查生成 `full.csv` 的脚本，修正 z 计算并做裁剪/winsorize，必要时重新生成 `processed_data/2/full.csv`。
2. **重跑 probe**：`python -m model.baseline_ovulation_probe full`，确认 `Feature mean |x|` 降到约 0.5～1，再看 cycle accuracy 与 AUC。
3. 若仍不理想，再考虑：增加/清洗样本、加入周期/先验特征、或引入领域知识约束。

（Probe 内已加特征 clip，可在未改数据前先看“修掉极值后”的大致趋势。）
