# 数据处理改进方案 v2

> 依据：Hamidovic et al. 2023（HRV 与排卵期）、Wang et al. 2025（Apple 腕温算法）及当前 pipeline 分析。

---

## 一、改进背景与核心问题

### 1.1 两篇论文的关键结论

| 论文 | 信号 | 关键结论 |
|------|------|----------|
| Hamidovic 2023 | HF-HRV（睡眠/晨间静息） | 围排卵期（LH 峰附近 3 天）HF-HRV **均值整体下降**，相邻相位差异显著；**个体间方差占 68.86%**，信号是相位级而非单日尖峰 |
| Wang 2025（Apple） | 整夜腕温（sleeping wrist temp） | 排卵后孕酮驱动温度**持续上升 ≥0.2°C**，构成双相转折；算法**追溯性**等转折出现后再推算排卵日，延迟中位数 4 天；约 **20% 周期无有效温度信号** |

### 1.2 当前 pipeline 与文献的错配

| 错配点 | 当前做法 | 文献对齐做法 |
|--------|----------|-------------|
| 主要输入文件 | `full.csv`（全天特征）| 应为 `sleep.csv`（整夜睡眠期，与两篇论文对齐）|
| 缺少周期位置先验 | 无「周期内第几天」特征 | Wang 2025 以「上次月经第一天」为输入；Hamidovic 以 LH 峰对齐各相位 |
| 腕温特征类型 | 单夜 mean/std/min/max | 应检测**跨日双相转折**（多日趋势/斜率）|
| 缺失值填充 | 被试全局均值 → 0 | 应使用**邻日插值**，避免抹平相位差异 |
| 滚动 z 基线 | K=14 天滚动，跨越排卵事件 | 基线应来自**当前周期早期（月经期/卵泡早期）**，不跨排卵 |
| morning 窗口 HRV | 仅 10 条数据，等同无效 | 放弃 morning 作为 HRV 窗口；sleep 窗口是有效替代 |

---

## 二、改进方案（按优先级）

---

### P1（必做）：加入周期位置特征 `day_in_cycle`

**改动位置**：`daily_data_2.ipynb` STEP 4（合并后）或 `dataset.py` 的 `load_and_merge`

**具体做法**：

在 `cycle_clean_2.csv` 与日级特征合并后，按 `(id, study_interval, small_group_key)` 分组，计算：

```
day_in_cycle = day_in_study - min(day_in_study)  # 每个周期从 0 开始
```

同时加归一化版本：

```
day_in_cycle_norm = day_in_cycle / max(day_in_cycle)  # ∈ [0, 1]，表示当前处于周期哪个比例位置
```

**理由**：
- Wang 2025 明确将「上次月经第一天」作为算法输入；
- Hamidovic 2023 的显著效应本质是「处于哪个周期相位」的函数；
- 没有这个信息，模型无法利用「排卵多发生在周期 40–60% 位置」的基础先验；
- 这是**成本最低、收益最高**的改动。

**注意**：`day_in_cycle` 和 `day_in_cycle_norm` 是结构性先验，**不做 z 标准化**，直接作为原始值输入（模型可自行学习其标度）。

---

### P2（必做）：将主要输入从 `full.csv` 改为 `sleep.csv`

**改动位置**：`config.py`

```python
SLEEP_CSV = os.path.join(WORKSPACE, "processed_data", "2", "sleep.csv")
```

同步更新 `FEATURE_COLS` 不变（列名相同），只改文件路径。

**理由**：
- HRV（RMSSD）：Fitbit 仅在睡眠期间测量，sleep 窗口数据量充足（4042 天 vs morning 仅 10 天）；Grant et al. 用睡眠 RMSSD 发现转折；
- 腕温（wt_mean 等）：Apple 算法明确使用 overnight sleeping wrist temperature；白天温度受活动、环境、饮食扰动大，睡眠期更稳定；
- 心率：睡眠心率最接近静息状态，受日间活动干扰最小。

---

### P3（必做）：修改缺失值填充策略

**改动位置**：`daily_data_2.ipynb` STEP 5

**当前做法**：
```python
subj_mean = out.groupby(["id", "study_interval"])[col].transform("mean")
out.loc[miss, col] = subj_mean  # 用被试整体均值填充
out.loc[out[col].isna(), col] = 0
```

**改为**：线性插值（在同一被试的观测期内，按 `day_in_study` 插值）+ 剩余填 0：

```python
# 先按 day_in_study 线性插值（仅在有值的两侧做插值，不外推）
out = out.sort_values(["id", "study_interval", "day_in_study"])
out[col] = out.groupby(["id", "study_interval"])[col].transform(
    lambda x: x.interpolate(method="linear", limit_direction="both", limit=3)
)
# 超出插值范围（连续缺失超过 3 天）填 0
out.loc[out[col].isna(), col] = 0
```

**参数说明**：
- `limit=3`：最多插值补 3 天的连续缺失，超过则填 0（避免长段缺失被不真实地填满）；
- `limit_direction="both"`：允许向两侧插值。

**理由**：
- 邻日插值保留了「相邻天生理量相近」的时序结构，不会把卵泡期均值填入黄体期；
- Hamidovic 2023 显示相位间差异显著，全局均值填充会将不同相位的值混合，消除了相位信号；
- 缺失指示符（`*_missing`）已保留，模型可自行学习「缺失 vs 有效」的区别。

### P4（必做）：修改 rolling z 归一化基线定义

**改动位置**：`daily_data_2.ipynb` STEP 6

**核心问题**：当前 K=14 天窗口会在排卵后（黄体期）把排卵前后混在一起作为基线，抹平了排卵期的温度上升信号。

**改为两段式归一化**：

```python
K_CYCLE_EARLY = 5   # 当前周期前 N 天作为周期内基线
K_LONG = 28         # 长期窗口：估计个体稳定水平（用于历史无数据时的备用）
```

- **优先**：对每个 `(id, small_group_key)` 周期，用**周期内前 K_CYCLE_EARLY 天（月经期）的均值和标准差**作为该周期的基线，对周期内所有天做 `z = clip((x - μ_early) / (σ_early + ε), ±5)`；
- **备用**：若周期内前 5 天数据缺失超过 60%，则回退使用 K_LONG=28 天的滚动基线（即修复后的当前方案）；
- 不论哪种方式，均保留 `Z_CLIP = 5`。

**理由**：
- Hamidovic 2023 的对比基准是「卵泡早期（月经结束后 1–4 天）」，周期内早期归一化与之对齐；
- Apple 算法的核心是检测温度相对于卵泡期基线是否上升；K=14 滚动会在黄体期把黄体早期温度也纳入基线，削弱检测能力；
- 第一周期的前 5 天方差可能为 0（数据少），故设 σ < ε 时 z=0（沿用现有处理）。

---

### P5（建议）：新增腕温双相转折衍生特征

**改动位置**：`daily_data_2.ipynb` STEP 3 之后、STEP 6 之前（在已完成聚合且填充缺失后）

新增两个衍生特征（**在 z 标准化之前**计算，单位为 °C）：

```python
# wt_shift_7v3: 近 3 天腕温均值 - 再过去 4 天均值（检测近期是否有持续上升）
# 正值 = 近日温度相对更近的过去升高；>=0.2 对应 Apple 算法的阈值
wt_recent = rolling_mean(wt_mean, window=3, min_periods=1)          # 近 3 天均值
wt_prior  = rolling_mean(wt_mean, window=7, min_periods=3).shift(3) # 前 3–9 天均值
wt_shift_7v3 = wt_recent - wt_prior                                 # 双相转折指标
```

同理对 `nightly_temperature` 做同样计算，得到 `temp_shift_7v3`。

这两个特征反映「是否已经发生了双相转折」，之后再做 z 归一化（但因为已是差分量，z 归一化的必要性降低，可选保留原始值）。

**理由**：Apple 算法的核心信号是检测 ≥0.2°C 的持续温度上升；单日 `wt_mean` 无法承载这个信息；差分特征可以明确告知模型「今日相比近期基线温度是否已升高」。

### P6（可选）：重新评估 LF 和 LF/HF ratio

Hamidovic 2023 明确指出：LF 作为交感活动指标「存在争议」，论文本身仅聚焦 HF-HRV。

**建议**：
- 保留 `lf_mean_z` 和 `lf_hf_ratio_z`，但在 probe 阶段的消融实验中，单独测试去掉这两列对 AUC 的影响；
- 若去掉后 AUC 没有明显下降，可在后续版本中简化特征集。

## 六、预期效果与基准

| 改动 | 预期对 probe AUC 的影响 | 预期对主模型的影响 |
|------|------------------------|-------------------|
| P1 day_in_cycle | **明显提升**（模型能利用周期结构先验，HRV/温度特征更易区分相位）| 主任务 MAE 下降，尤其 horizon 21+ 改善 |
| P2 sleep.csv | 小幅提升（HRV/温度更干净，全天窗口的活动噪声消除）| 略微改善 |
| P3 邻日插值 | 小幅提升（相位信号不被全局均值模糊）| 略微改善 |
| P4 周期内基线 z | **中等提升**（尤其对腕温温度双相转折的检测更敏感）| 排卵辅助头可能改善 |
| P5 双相转折特征 | **潜在较大提升**（直接提供「已发生温度转折」的信号）| ovulation_corr 可能改善 |

即便所有改动后 probe AUC 仍不高（如 0.55–0.65），也不代表失败——文献（Wang 2025）表明约 20% 周期根本没有有效温度信号，存在天花板；关键是确认「改进后的信号质量」相比 v1 是否有提升。
