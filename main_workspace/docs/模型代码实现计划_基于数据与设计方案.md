# 模型代码实现计划（基于 cycle_clean_2、processed_data/2 与设计方案）

本文档结合当前数据（`cycle_clean_2.csv`、`processed_data/2`）与论文中的**序列潜在状态多任务模型（sequential latent-state modelling）**设计，整理一套**模型代码实现计划**，仅做分析与整理，不包含具体代码。方案已按论文公式与训练协议逐项对齐并细化。

## 零、论文模型与公式速查（实现对照）

| 论文表述           | 公式 / 约定                                                                                                     | 实现对应                                                                                                                                                 |
| ------------------ | --------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 序列日级输入       | \(\{x_1,x_2,\ldots,x_t\}\)，经生理聚合与**个体内标准化**                                                  | `processed_data/2` 的日级特征，仅用 `*_z` 列，**不包含 lh/estrogen**                                                                           |
| 循环状态更新       | \(h_t = f(x_t, h_{t-1})\)，\(f\) 为循环更新函数                                                                 | **GRU** 为主结构，**LSTM** 为结构对比实验；同协议、同数据                                                                                    |
| 主任务：月经预测   | \(\hat{y}_t = W_m h_t + b_m\)，预测「距下次月经来潮的天数」                                                     | 回归头，输出标量；**time-to-event**，仅完整周期有监督；截断边界 mask                                                                               |
| 辅助任务：排卵推断 | \(p(t_{\text{ov}}) = \sigma(W_{\text{ov}} h_t + b_{\text{ov}})\)，二分类                                        | 仅在有**排卵标注**的时间步上计算 BCE，其余步 **mask 不参与梯度**                                                                             |
| 辅助损失           | \(\mathcal{L}_{\text{ov}} = \sum_{t \in \mathcal{T}_{\text{ov}}} \ell_{\text{BCE}}(p(t_{\text{ov}}), o_t)\) | **采用方案 B**：\(o_t \in [0,1]\) 概率软标签（ovulation_prob_fused），BCE 支持连续目标；\(\mathcal{T}_{\text{ov}}\) = 有排卵标注的周期内所有时间步 |
| 总损失             | \(\mathcal{L} = \mathcal{L}_{\text{menses}} + \lambda \mathcal{L}_{\text{ovulation}}\)                        | \(\lambda\) 为超参；辅助任务作为 **inductive bias**，不主导优化                                                                                    |
| 两阶段训练         | Stage 1：骨干 + 双头联合优化；Stage 2：主任务微调，骨干冻结或低学习率                                           | 避免稀疏/噪声辅助监督过度影响主任务                                                                                                                      |

## 一、当前数据与模型设计的对应关系

### 1.1 数据资产概览

| 数据源                                                          | 主键/粒度                                                     | 内容概要                                                                                                                                                              | 在模型中的角色                                                                                                                                |
| --------------------------------------------------------------- | ------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **cycle_clean_2.csv**                                     | (id, study_interval, day_in_study)，按 small_group_key 成周期 | 周期内每日：phase、lh/estrogen、ovulation_day_method1/2、ovulation_prob_fused                                                                                         | **标注与序列划分**：提供周期边界、排卵标签、月经来潮日；**激素仅用于标注与评估，不进入模型输入**（wearable-only sensing，防泄漏） |
| **processed_data/2/index.csv**                            | (id, study_interval, day_in_study)                            | 日级索引（与 cycle_clean_2 行键一致，约 4825 行）                                                                                                                     | 与 cycle 对齐时确定「哪些日」在周期内                                                                                                         |
| **processed_data/2/full.csv**（及 morning/evening/sleep） | 同上                                                          | 可穿戴日级特征：HRV(rmssd, lf, hf, lf_hf_ratio)、HR(mean, std, min, max)、腕温(wt_*)、nightly_temperature、resting_hr；含\*_missing 与 **\*_z（个体内标准化）** | **模型输入 x_t**：多模态日级特征，已做 within-individual 标准化，无激素                                                                 |

**数据规模（供实现参考）**：

- `cycle_clean_2.csv`：约 4825 行，每行对应「某周期内某日」；`full.csv` 行数多于 cycle（约 11714 行），需以 cycle_clean_2 的 (id, study_interval, day_in_study) 为键从 full 中取子集，保证只使用**周期内**的日级特征。
- 序列按 `small_group_key` 分组、按 `day_in_study` 升序排列，得到 \(\{x_1,\ldots,x_T\}\)；主任务标签在 cycle 表上推导，辅助任务由排卵相关列得到并严格按 \(\mathcal{T}_{\text{ov}}\) 做 mask。

### 1.2 特征列与输入维度

- **full.csv** 中可用于 \(x_t\) 的列（仅可穿戴、已标准化）：`rmssd_mean_z`, `lf_mean_z`, `hf_mean_z`, `lf_hf_ratio_z`, `hr_mean_z`, `hr_std_z`, `hr_min_z`, `hr_max_z`, `wt_mean_z`, `wt_std_z`, `wt_min_z`, `wt_max_z`, `nightly_temperature_z`, `resting_hr_z`。共 14 维；若加入 morning/evening/sleep 的对应 z 或原始日级指标，维度可扩展。
- **缺失**：full 中已有 `*_missing` 列，实现时需约定缺失策略（如填 0、mask 向量、或单独 missing 特征），与论文「稀疏、异质观测」一致。
- **异常值**：若发现 `*_z` 列存在极端值（如上游标准化缺陷），需在数据层做 clip 或回溯至数据处理阶段修正，避免破坏训练稳定性。

### 1.3 周期与监督信号在 cycle_clean_2 中的推导

- **周期边界**：每个 `small_group_key` 对应一个完整周期；周期内首日为该周期「月经开始日」，下一周期首日即为「下一次月经开始日」。
- **主任务标签 days_until_next_menses**：对周期内每个 day_in_study，取「下一次月经开始日」（即下一周期首日的 day_in_study），\(y_t = \text{next\_menses\_day\_in\_study} - \text{day\_in\_study}\)。周期末尾若干日若无法观测到下次月经来潮（截断），则主任务在该时间步 **mask**，不参与 \(\mathcal{L}_{\text{menses}}\)（与论文 “masked for truncated cycle boundaries where future onset information is unavailable” 一致）。
- **排卵标签（辅助任务）**：**采用方案 B**，使用 **ovulation_prob_fused** 作为 \(o_t \in [0,1]\) 概率软标签，BCE 损失；详见 **2.5**。\(\mathcal{T}_{\text{ov}}\) = 有排卵标注的周期内所有时间步；无排卵标注的周期整周期不参与辅助损失。
- **排卵 mask 实现**：对每个时间步维护 `ovulation_mask`（True/False）。仅当 `ovulation_mask=True` 时该步参与 \(\mathcal{L}_{\text{ovulation}}\)（BCE）；否则该步对辅助损失贡献为 0，不参与梯度。

## 二、模型代码实现计划（模块化）

### 2.1 数据层（Data Pipeline）

| 步骤 | 内容                                           | 输入                                                                   | 输出/约定                                                                                                                                                                       |
| ---- | ---------------------------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| D1   | 周期内日集合与周期元信息                       | cycle_clean_2.csv                                                      | 每个 small_group_key 的 (id, study_interval, day_in_study 列表)、周期长度、next_menses_day_in_study                                                                             |
| D2   | 日级特征加载与对齐                             | processed_data/2/full.csv（及可选 morning/evening/sleep）、D1 的日集合 | 仅保留「周期内日」的行；列 = 标识 + 14 维 z（+ 可选扩展）+\*_missing；缺失策略统一（如填 0 + missing 指示）                                                                     |
| D3   | 主任务标签与截断 mask                          | D1 的 next_menses_day_in_study、周期边界                               | 每行增加 days_until_next_menses；**mask_menses**：截断边界（无未来月经信息）为 False，不参与 \(\mathcal{L}_{\text{menses}}\)                                              |
| D4   | 辅助任务标签与\(\mathcal{T}_{\text{ov}}\) mask | **ovulation_prob_fused**（方案 B）                               | 每行增加 ovulation_label\(o_t \in [0,1]\)（概率软标签）与 **ovulation_mask**（True = 该步在 \(\mathcal{T}_{\text{ov}}\) 内）；无排卵标注的周期整周期 ovulation_mask=False |
| D5   | 序列构造                                       | 按 (id, small_group_key) 分组、按 day_in_study 升序                    | 每条样本 = 一个周期序列：X=[x_1..x_T], y_menses, y_ov, mask_menses, mask_ov；**每序列对应一个周期，推理时 \(h_0=\mathbf{0}\)**；train/val/test 按 subject 划分            |

**注意**：

- 与 cycle_clean_2 对齐时，以 cycle_clean_2 的 (id, study_interval, day_in_study) 为右表做 join，保证只保留完整周期内的日。
- processed_data/2 的 index 可能包含周期外日，需过滤；序列长度 T 因周期而异，需支持变长或 padding + length 信息。

### 2.2 特征与输入规范

| 项目            | 约定                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| 输入特征\(x_t\) | 仅来自 processed_data/2，不包含 lh、estrogen 及任何激素列              |
| 标准化          | 已由上游数据处理完成（\*_z）；模型侧仅做可选的全局 scale 或不再变换    |
| 缺失            | 使用\*_missing 或统一填 0；若用 mask，需在损失/前向中屏蔽对应位置      |
| 序列顺序        | 严格按 day_in_study 升序，保证\(h_t = f(x_t, h_{t-1})\) 的时间方向正确 |

### 2.3 模型结构（与论文公式一一对应）

| 模块                         | 论文公式/职责                                                                                      | 实现要点                                                                                                                                                                                                                                                                                                         |
| ---------------------------- | -------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **循环骨干**           | \(h_t = f(x_t, h_{t-1})\)（式 3.3）                                                                | **主结构**：GRU；**对比结构**：LSTM，同数据、同协议。输入 dim = 特征维（如 14 维 z）；hidden_size 超参；建议先单层（论文强调 parameter-efficient、small-sample）。**初始状态 \(h_0\)**：每个周期序列**独立**，\(h_0 = \mathbf{0}\)（跨周期不延续），保证周期内时序一致、避免跨周期泄漏。 |
| **月经头（主任务）**   | \(\hat{y}_t = W_m h_t + b_m\)（式 3.4）                                                            | Time-to-event 回归，输出标量「距下次月经天数」。**损失**：仅对「有真实标签且非截断边界」的时间步计算（如 MSE 或 L1）；截断边界步 mask 不参与。                                                                                                                                                             |
| **排卵头（辅助任务）** | \(p(t_{\text{ov}}) = \sigma(W_{\text{ov}} h_t + b_{\text{ov}})\)（式 3.5）                         | 二分类，输出概率。**损失**：\(\mathcal{L}_{\text{ov}} = \sum_{t \in \mathcal{T}_{\text{ov}}} \ell_{\text{BCE}}(p(t_{\text{ov}}), o_t)\)（式 3.6）；实现时仅对 `ovulation_mask=True` 的步计算 BCE 并聚合，其余步贡献为 0、不反传。                                                                    |
| **总损失**             | \(\mathcal{L} = \mathcal{L}_{\text{menses}} + \lambda \mathcal{L}_{\text{ovulation}}\)（式 3.7） | \(\lambda\) 控制辅助任务权重；辅助任务作为 **inductive bias**，不主导优化。                                                                                                                                                                                                                                |

**整体结构（对应论文 Figure 3.1）**：共享的循环骨干输出 \(h_t\)，其上接两个任务头（月经回归头、排卵分类头），形成 sequential multi-task latent cycle state model。
**设计动机（与论文一致）**：循环结构显式建模**渐进状态演化**与时间依赖，在小样本下保持参数效率与计算可承受；不采用 Transformer/Mamba 等，因其优势在大规模、长序列下更明显。

### 2.4 两阶段训练协议（论文明确要求）

| 阶段              | 内容                                                                                                                               | 说明                                                                                                                                        |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Stage 1** | 联合优化：循环骨干 + 月经头 + 排卵头，优化\(\mathcal{L} = \mathcal{L}_{\text{menses}} + \lambda \mathcal{L}_{\text{ovulation}}\) | 学习率、epoch、early stop 等超参；双任务同时监督，使\(h_t\) 同时服务于主任务与排卵的弱监督。                                                |
| **Stage 2** | 主任务微调：**仅**优化月经头（或月经头 + 骨干低学习率）；排卵头不再参与损失                                                  | 骨干可**frozen** 或 **低学习率** 更新。目的：辅助监督稀疏且可能带噪，过度耦合可能干扰主任务；二阶段解耦以优先保证月经预测表现。 |

**数据划分**：见下节 2.4.1 的详细约定。

#### 2.4.1 训练/验证/测试划分与防泄漏（含是否采用 n-fold）

**泄漏来源与原则**

- **Subject 泄漏**：若同一人（id）的周期同时出现在训练集和测试集，模型可能学到该人的个体模式，测试集表现会高估泛化到**新用户**的能力。
- **时间泄漏**：按「日期」或「周期序号」随机划分而忽略 id，同一人的早期周期在 train、后期在 test，仍属同一 subject 泄漏。
- **原则**：划分按 **subject (id)** 进行，保证 **同一 id 的所有周期只属于 train、val 或 test 之一**，三者互不重叠。

**推荐划分方式**  

| 方式                              | 做法                                                                                                                                                  | 适用场景                                                               |
| --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| **留出法（hold-out）**      | 按 id 划分为 train / val / test（如 70% / 15% / 15% 的 subject 数）；每折内该 id 下所有周期进入同一集合。                                             | subject 数较多（如 >30）时，可固定一次划分，便于复现。                 |
| **K-fold 按 subject**       | 将**subject 列表** 均分为 K 份；第 k 折：第 k 份 subject 为 val（或 test），其余为 train。**以 subject 为折叠单位，不以周期为折叠单位**。 | subject 数有限时，得到 K 次不同的 train/val，报告均值±标准差，更稳。  |
| **留一 subject 法（LOSO）** | 每折留 1 个 subject 作 val（或 test），其余全部作 train；共 n 折（n = subject 数）。                                                                  | subject 很少（如 <15）时，最大化训练数据，但每折 val 仅 1 人，方差大。 |

**是否需要 n-fold（K-fold）**

- **需要**：在**小队列**下，subject 数有限，单次划分的 test 结果波动大；用 **subject-level K-fold** 可得到多次 val 估计，报告 **mean ± std**，更可靠。

**选定方案：固定 test + K-fold 仅作 train/val**

- **步骤**：
  1. 先按 id 去重得到 subject 列表；**留出少量 subject（如 2–3 人）作为固定 test 集**，全程不参与训练与超参选择。
  2. **其余 subject** 做 **K-fold（如 5-fold）仅用于 train/val**：每折中，第 k 份 subject 作 val，其余作 train；用于模型选择、早停、超参（如 \(\lambda\)、学习率）调优。
  3. **最终评估**：在**固定 test 集上只评估一次**，报告主任务与辅助任务指标，作为泛化到「未见过 subject」的估计。
- **优点**：test 集完全独立、评估可复现；K-fold 仅作用于 train/val，避免在 test 上多次评估导致间接过拟合。
- 每折内：样本单位是「周期序列」，周期归属 train/val 由其 **id 属于哪一折** 决定。

**检查清单（防泄漏）**

- [ ] 划分键是 **id**（或 id + study_interval，若同一人多年视为不同「用户」），不是 cycle_id、不是 day_in_study。
- [ ] 任意一个 id 不能同时出现在 train 与 val，或 train 与 test，或 val 与 test。
- [ ] 若使用 K-fold，每一折的 train/val 均满足上述条件。

### 2.5 排卵标签：采用方案 B（概率软标签）

**选定方案：方案 B（概率作为软标签）**

- 直接使用数据中的概率作为目标：\(o_t \in [0,1]\)（如 **ovulation_prob_fused**），不做二值化。
- 损失：**BCE**，对软标签天然支持：\(\ell_{\text{BCE}}(p_t, o_t) = -[o_t \log p_t + (1-o_t)\log(1-p_t)]\)，其中 \(o_t\) 为连续值。
- \(\mathcal{T}_{\text{ov}}\)：有排卵标注的周期内，所有时间步参与；概率高的日对损失贡献更大。
- 优点：保留内分泌不确定性、贴合「排卵窗口」的生理模糊性；与现有数据格式一致。
- 报告时可注明：辅助任务为「排卵概率软分类」，损失为 BCE，目标为 [0,1]。

**方案 A（二值，可选消融）**

- 每周期取一天（如 argmax）为 \(o_t=1\)，其余为 0；与论文 binary 字面一致。
- 消融实验中可与方案 B 对比，比较对主任务与排卵头的影响。

### 2.6 月经预测任务的语义：逐日更新、预测含义与「正确性」相对该周期

- **输入与更新**：每个**周期**对应一条序列 \(\{x_1,\ldots,x_T\}\)，按 `day_in_study` 升序。模型从 \(h_0=\mathbf{0}\) 开始，**随每日数据输入**依次更新：\(h_1=f(x_1,h_0),\,h_2=f(x_2,h_1),\ldots\)。
- **预测含义**：在**每个时间步 t**，模型输出 \(\hat{y}_t = W_m h_t + b_m\)，表示「**从第 t 天起，距下次月经来潮的天数**」（time-to-event）。同一周期内 \(\hat{y}_1,\hat{y}_2,\ldots\) 随信息增加而逐日更新。
- **真实值 \(y_t\) 的定义**：在**该周期内**，取该周期的「下次月经来潮日」（即下一周期首日的 day_in_study），\(y_t = \text{next\_menses\_day} - \text{day\_in\_study}(t)\)。因此 \(y_t\) 由**该周期的长度/该周期的下次月经日**唯一确定。
- **预测是否正确：基于该周期**
  - **距下次月经天数是否预测正确，是基于该周期的真实长度（即该周期对应的 next_menses 日期）来判定的**。
  - 即：在同一周期内，真实值 \(y_t\) 与预测值 \(\hat{y}_t\) 均在「该周期」的语境下定义；我们比较的是「模型预测的剩余天数」与「该周期实际的下次月经日推算出的剩余天数」，不做跨周期混用。
  - 评估时：按 (周期, 时间步) 逐点比较 \(\hat{y}_t\) 与 \(y_t\)；可选补充**相对误差**（如 MAE / 该周期长度、或 MAPE），以体现对长周期与短周期的公平性。
- **训练**：对有标签且非截断的每个 (t, 周期) 计算回归损失（如 L1/MSE），\(y_t\) 均来自该周期的 next_menses_day。

### 2.7 评估与指标

| 任务     | 指标                                                         | 说明                                                                                                          |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------- |
| 月经预测 | MAE（或中位数 AE）；可选 MAE/周期长度、±1/2/3 天准确率      | 预测 vs 真实「距下次月经天数」；**正确性基于该周期真实长度（该周期 next_menses 日）**；可按预测视野分层 |
| 排卵推断 | 二分类 F1、Accuracy；±1 天容忍；若用软标签可加相关系数/校准 | 仅在有排卵标注的周期/日上评估；可与 method1/method2 分别对齐                                                  |

**边界与 mask**：

- 周期末「无未来月经信息」的时间步：主任务不计算损失（mask）；
- 无排卵标注的周期：辅助任务整周期不参与损失与评估。

### 2.8 实验与消融（可选）

- **结构对比**：GRU vs LSTM，同数据、同协议（论文 “LSTM trained as a structural comparator under identical protocols”）。
- **辅助任务权重**：不同 \(\lambda\)（含 \(\lambda=0\) 即无排卵头）。
- **排卵标注来源**：method1 vs method2（或 fused）对主任务与排卵头的影响。
- **排卵标签形式**：二值（单日 argmax）vs 概率软标签（BCE 目标为 [0,1]），对主任务与排卵头的影响（见 2.5）。
- **缺失/稀疏**：随机 mask 部分日级特征或部分时间步，观察鲁棒性（与论文「稀疏、异质观测」呼应）。

### 2.9 实现与数据检查清单（防错）

- [ ] **输入**：确认 \(x_t\) 仅包含 processed_data/2 的 z 列，**无** lh、estrogen 或任何激素列。
- [ ] **主任务 mask**：截断边界（无未来月经 onset 的时间步）不参与 \(\mathcal{L}_{\text{menses}}\)。
- [ ] **辅助任务 mask**：仅 \(\mathcal{T}_{\text{ov}}\) 内时间步参与 BCE；无排卵标注周期整周期不参与。
- [ ] **序列与初始状态**：按 (id, small_group_key) 一组、day_in_study 升序；每序列 \(h_0=\mathbf{0}\)，跨周期不延续。
- [ ] **两阶段**：Stage 2 仅优化主任务相关参数，骨干冻结或低学习率；排卵头不再参与损失。
- [ ] **划分与防泄漏**：**固定 test**（留出少量 subject）+ **K-fold 仅作 train/val**；划分键为 **id**，同一 id 不跨集合（见 2.4.1）。
- [ ] **排卵标签**：辅助任务采用 **方案 B**，\(o_t\) 为 ovulation_prob_fused 概率软标签，BCE 损失。
- [ ] **主任务评估**：预测正确性基于**该周期**的真实长度（该周期 next_menses 日）；评估时按周期内逐步比较。
- [ ] **评估**：排卵评估仅在有排卵标注的周期/日上进行；激素仅用于标注与评估，不参与训练输入。

## 三、与论文设计的一一对应（自查用）
