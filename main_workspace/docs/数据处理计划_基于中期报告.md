# 基于中期报告的数据处理分析与计划

本文档仅依据**中期报告**与 **mcPHASES 数据集文档**，分析为达成报告中的方法所需的数据处理内容、目标数据结构，并给出可执行的数据处理计划。不依赖 open_source 内容。

---

## 一、中期报告对数据与处理的要求（摘要）

### 1.1 数据来源与用途

- **主数据**：mcPHASES（PhysioNet），42 名参与者，两期研究（2022 / 2024）。
- **周期与排卵**：以**排卵**为时间锚点；自然周期定义为**连续两次月经开始日之间**；仅保留**边界清晰、完整**的周期。
- **排卵日标注**：基于每日尿 LH（及雌激素）的 surge 规则，操作定义可为「surge 开始 +1 天」「surge 峰值 +1 天」「最后一次明显峰值 +1 天」等，用于标注与消融，**不**作为模型输入。
- **模型输入**：**仅可穿戴衍生的日级特征**；**禁止**将 LH、PdG、E3G 等激素放入特征，以模拟真实可穿戴部署并避免泄漏。

### 1.2 生理特征（报告 3.2 节）

- **HRV**：睡眠期间、约 5 分钟分辨率 → 聚合到**日级**；排除覆盖不足或质量差的夜晚。
- **WST（腕部皮肤温度）**：相对个人基线的偏差等 → **日级**聚合（主睡眠或预定义夜间窗口的稳健统计）。
- **标准化**：**仅用该个体历史**的滚动基线做 within-individual 标准化，避免时序泄漏：
  - \( z_t = (x_t - \mu_t) / (\sigma_t + \epsilon) \)
  - \( \mu_t, \sigma_t \) 来自 \( t \) 之前 \( K \) 天；不足 \( K \) 天时用扩展窗口；\( \sigma_t \) 有下界以保证数值稳定。

### 1.3 其他模态与对齐

- **睡眠指标**：聚合到日级，与生理特征、周期锚点对齐。
- **自我报告症状**：日级，与生理特征、周期锚点对齐；保持原有标准化尺度。
- **输出**：所有模态按日对齐后，拼接成**统一的日级特征向量**，且**不包含任何激素测量**。

### 1.4 模型所需监督信号

- **主任务**：经期预测 → 每个 day 需要「**距下一次月经开始的天数**」（time-to-event）。
- **辅助任务**：排卵推断 → 仅在**有排卵标注**的日有 0/1 标签；其余日 mask，不参与辅助损失。

## 三、mcPHASES 中相关表与连接方式

以下均以 **`(id, study_interval, day_in_study)`** 为日级主键与 `cycle_anchor_clean` 对齐（同一参与者在同一 study_interval 内 day_in_study 连续且唯一）。

| 数据表                         | 主键/时间粒度                                          | 与 day_in_study 的对应关系                                                                                                  |
| ------------------------------ | ------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| hormones_and_selfreport        | (id, study_interval, day_in_study)                     | 直接对应；含 phase, 自报症状；**激素列仅用于标注，不进入特征**                                                        |
| cycle_anchor_clean             | (id, study_interval, day_in_study)                     | 已过滤为 109 个完整周期内的行；含 ovulation_day_method1/2                                                                   |
| heart_rate_variability_details | (id, study_interval, day_in_study, timestamp)          | 已有 day_in_study；文档称睡眠期 5 分钟间隔；需按日聚合并做质量过滤（如 coverage）                                           |
| computed_temperature           | (id, study_interval, sleep_start_day_in_study, …)     | 每夜一条；用**sleep_start_day_in_study** 或 **sleep_end_day_in_study** 归属到「某一天」；需统一归属规则（见下） |
| sleep                          | (id, study_interval, sleep_start/end_day_in_study, …) | 每段睡眠一条；需归属到某一 day_in_study（建议：以 sleep_end_day_in_study 为「该睡眠所属日」）                               |
| sleep_score                    | (id, study_interval, day_in_study)                     | 已有 day_in_study，可直接按日连接                                                                                           |
| resting_heart_rate             | (id, study_interval, day_in_study)                     | 已有 day_in_study，每 participant 每日一行，可直接按日连接                                                                  |

**温度/睡眠归属约定（建议）**：

- **computed_temperature**：一夜一条，归属到 **sleep_end_day_in_study**（醒来那天），即「当天早晨结束的那一夜」的温度算作该日的夜间温度。
- **sleep**：同属 **sleep_end_day_in_study**；若一夜有多段，可只保留 main sleep 或按 duration 取主段，再聚合为日级睡眠特征。

### 3.1 可穿戴生理表详细说明（HR、HRV、腕温、computed temperature、resting heart rate）

以下五张表为可穿戴衍生的心率、HRV、腕部温度与计算温度，用于阶段 1 的日级特征提取。数据文件均位于 mcPHASES 数据集目录下。

| 数据表 | 文件 | 数据含义 | 主键 | 时间粒度 | 与 day_in_study 的对应关系 |
|--------|------|----------|------|----------|----------------------------|
| **heart_rate** | heart_rate.csv | 连续心率测量：Fitbit 记录的实时心率（bpm）及置信度（confidence）。数据量极大（约 6300 万行），为分钟级甚至更细的连续采样。 | (id, study_interval, day_in_study, timestamp) | 每条记录带时间戳，非固定间隔（样例约 5 秒一条）；按日有多条。 | **已有 day_in_study**。按 (id, study_interval, day_in_study) 分组后可做日级聚合（如睡眠窗口内均值、静息时段均值等）；若仅用静息心率可优先用 resting_heart_rate 表。 |
| **heart_rate_variability_details** | heart_rate_variability_details.csv | 心率变异性详情：睡眠期间 HRV，含 rmssd（时域）、low_frequency（LF）、high_frequency（HF）及 coverage（数据覆盖率）。用于自主神经与睡眠质量相关特征。 | (id, study_interval, day_in_study, timestamp) | **5 分钟间隔**，且主要在睡眠时段；同一 (id, study_interval, day_in_study) 下有多条（多时间戳）。 | **已有 day_in_study**。按 (id, study_interval, day_in_study) 分组后对 rmssd/LF/HF 做稳健聚合（如均值、中位数），并按 coverage 做质量过滤（剔除覆盖不足的夜）。 |
| **wrist_temperature (WT)** | wrist_temperature.csv | 腕部皮肤温度原始序列：相对个人基线的温度偏差（°C），即 temperature_diff_from_baseline。为连续时间序列。 | (id, study_interval, day_in_study, timestamp) | **1 分钟间隔**；同一 (id, study_interval, day_in_study) 下有多条（全天或夜间）。 | **已有 day_in_study**。需在「主睡眠窗口」或预定义夜间窗口内做稳健统计（均值、标准差等）得到日级 WST 特征；可与 sleep 表对齐确定睡眠时段再聚合。 |
| **computed_temperature** | computed_temperature.csv | 计算的夜间温度：对单次睡眠会话预聚合的指标。每夜一条，含 nightly_temperature、baseline_relative_* 等。type 多为 SKIN。 | (id, study_interval, sleep_start_day_in_study, sleep_start_timestamp) 或按「一夜」唯一；表中无 day_in_study 列。 | **每夜一条**（一条 = 一段睡眠会话）。 | **无 day_in_study**。用 **sleep_end_day_in_study** 作为该夜温度的「归属日」（醒来那天）；即该夜算作 sleep_end_day_in_study 这一天的夜间温度。与 cycle_anchor_clean 对齐时用 day_in_study = sleep_end_day_in_study。 |
| **resting_heart_rate** | resting_heart_rate.csv | 每日静息心率：Fitbit 估计的当日静息心率（value，bpm）及估计误差（error）。为日级汇总，可直接作特征。 | (id, study_interval, day_in_study) | **日级**，每 (id, study_interval, day_in_study) 一行。 | **已有 day_in_study**。与 cycle_anchor_clean 或日级特征表直接按 (id, study_interval, day_in_study) 连接即可。 |

**列名速查**：

- **heart_rate**：id, study_interval, is_weekend, day_in_study, timestamp, bpm, confidence  
- **heart_rate_variability_details**：id, study_interval, is_weekend, day_in_study, timestamp, rmssd, coverage, low_frequency, high_frequency  
- **wrist_temperature**：id, study_interval, is_weekend, day_in_study, timestamp, temperature_diff_from_baseline  
- **computed_temperature**：id, study_interval, is_weekend, sleep_start_day_in_study, sleep_start_timestamp, sleep_end_day_in_study, sleep_end_timestamp, type, temperature_samples, nightly_temperature, baseline_relative_sample_sum, baseline_relative_sample_sum_of_squares, baseline_relative_nightly_standard_deviation, baseline_relative_sample_standard_deviation  
- **resting_heart_rate**：id, study_interval, is_weekend, day_in_study, value, error  

**与 day_in_study 对齐小结**：HR、HRV、WT、resting_heart_rate 均含 day_in_study，可按日聚合或直接连接；computed_temperature 需用 sleep_end_day_in_study 映射为 day_in_study 后再与周期锚点对齐。

## 四、目标数据结构（要达到的产出）

### 4.1 日级多模态特征表（模型输入用，无激素）

**表名建议**：`daily_features.csv` 或等价的按 (id, study_interval, day_in_study) 一张表。

**主键**：`id`, `study_interval`, `day_in_study`。

**列构成（概念）**：

- **标识**：`id`, `study_interval`, `day_in_study`。
- **周期结构（仅用于划分序列与评估，可放同一表或单独表）**：`cycle_id`（如 `id_study_interval_cycle_index`）、`day_in_cycle`（周期内第几天）、`phase`（Menstrual/Follicular/Fertility/Luteal）— 若 phase 来自数据集标注且非激素推导，可保留。
- **HRV 日级特征**（标准化后）：例如 `rmssd_z`, `hf_hrv_z`, `lf_hrv_z` 等（命名可带 `_z` 表示已标准化）；缺失日可为 NA。
- **WST 日级特征**（标准化后）：例如 `nightly_temperature_z`, `baseline_relative_nightly_std_z` 等；缺失为 NA。
- **睡眠日级特征**（可选标准化或保留原始尺度）：如 `minutesasleep`, `efficiency`, `deep_minutes` 等；缺失为 NA。
- **自报症状**（保留原有尺度或简单标准化）：如 `appetite`, `cramps`, `fatigue`, …；缺失为 NA。
- **不包含**：`lh`, `estrogen`, `pdg` 及任何激素衍生列。

**标准化**：HRV、WST 等按报告要求做 **within-individual 滚动标准化**（只使用该 id 下该 study_interval 内、该 day 之前的观测），在特征表里存标准化后的值即可。

### 4.2 周期级标签与监督信号（与 4.1 可拼接或分开）

**表名建议**：`cycle_labels.csv` 或合并进 `daily_features` 的扩展列。

**目的**：为序列模型提供每个「日」的监督目标。

- **主键**：`id`, `study_interval`, `day_in_study`（与 4.1 一致）；且仅包含 **cycle_anchor_clean 中出现的 (id, study_interval, day_in_study)**（即 109 个周期内的日）。

**列**：

- **周期标识**：`cycle_id`, `day_in_cycle`（周期内第几天，从 1 或 0 起计）。
- **下一次月经开始日**（用于 time-to-event）：
  - 方案 A：存 `next_menses_day_in_study`（该周期结束日的 day_in_study），则「距下次经期天数」= `next_menses_day_in_study - day_in_study`。
  - 方案 B：直接存 `days_until_next_menses`（非负整数）。
    二者等价，任选其一即可。
- **排卵标签（辅助任务）**：`ovulation_label`（0/1），仅在有排卵标注的日为非空；无标注日为 NA（训练时 mask）。
- 可选：保留 `ovulation_day_method1`, `ovulation_day_method2` 以便消融（同一行上通常仅对应日为 1，其余为 0）。

**注意**：同一周期内「下一次月经开始日」对所有天是常数；排卵标签仅在标注为排卵日的那一天为 1。

### 4.3 最终用于训练的「单表」视图（可选物理表或视图）

- 由 **4.1** 与 **4.2** 按 `(id, study_interval, day_in_study)` 外连接得到。
- 每一行 = 一个 (id, study_interval, day_in_study)，且该日属于 109 个完整周期之一。
- 列 = 标识 + 日级特征（仅可穿戴+睡眠+自报，无激素）+ `days_until_next_menses`（或等价）+ `ovulation_label`（0/1/NA）+ 可选 cycle_id / day_in_cycle / phase。
- 序列模型按 (id, study_interval, cycle_id) 分组，按 day_in_cycle 或 day_in_study 排序得到序列；mask 掉无排卵标注时间步的辅助损失。

## 五、数据处理计划（分步）

### 阶段 0：准备与周期锚点确认（可选）

- **0.1** 确认 `cycle_anchor_clean.csv` 中是否已有或需新增「周期 ID」列（如 `cycle_id` = 某大组内周期序号），以及每个周期内「下次月经开始日」是否可唯一确定（即该周期最后一天的 day_in_study 或下一周期第一天）。若无，在 cycle 表上增加：
  - `cycle_id`（如 `id_study_interval_cycle_index`），
  - `next_menses_start_day_in_study`（或等价：该周期结束日 +1 或下一周期首日）。
- **0.2** 若报告采用三种排卵定义，可保留 method1/method2 并增加 method3（last prominent peak +1），在标注表里统一三种 0/1 列，供后续消融。

### 阶段 1：可穿戴与睡眠的日级聚合

- **1.1 HRV 日级特征**

  - 读入 `heart_rate_variability_details.csv`，按 `(id, study_interval, day_in_study)` 分组。
  - 质量过滤：例如剔除 `coverage` 低于某阈值的 5 分钟块或整夜；保留的块内对 `rmssd`, `low_frequency`, `high_frequency` 做日级聚合（如均值、中位数、或稳健统计）。
  - 输出：每 (id, study_interval, day_in_study) 一行，列如 `hrv_rmssd_mean`, `hrv_hf_mean`, `hrv_lf_mean`（或等价命名）；缺失日不填或 NA。
- **1.2 WST / 夜间温度日级特征**

  - 读入 `computed_temperature.csv`，按「睡眠归属日」映射到 day_in_study：建议用 **sleep_end_day_in_study** 作为该夜温度的归属日；若存在同一夜多条（不同类型），可只保留 SKIN 或主类型。
  - 聚合到日：每 (id, study_interval, day_in_study) 至多一行，列如 `nightly_temperature`, `baseline_relative_nightly_standard_deviation` 等（若存在且非全空）；缺失为 NA。
  - 若使用 `wrist_temperature.csv` 原始时间序列：需先按 (id, study_interval) + 时间戳映射到 day_in_study（通过 sleep 或时间戳日期），再在「主睡眠窗口」内做稳健统计（如均值、标准差）得到日级 WST 特征；工作量大，可作后续扩展，优先用 computed_temperature 即可。
- **1.3 睡眠指标日级**

  - 读入 `sleep.csv`，按 **sleep_end_day_in_study** 归属到日；若一夜多段，可只保留 main sleep 或按 duration 取最长一段。
  - 按 (id, study_interval, day_in_study) 聚合：如 `minutesasleep`, `efficiency`, `minutestofallasleep`；可选解析 `levels` 得到 deep/light/rem 分钟数。
  - 读入 `sleep_score.csv`（已有 day_in_study），按 (id, study_interval, day_in_study) 与上面睡眠表合并或单独加入，列如 `overall_score`, `deep_sleep_in_minutes`, `restlessness` 等。

### 阶段 2：与周期锚点对齐、仅保留完整周期内的日

- **2.1** 以 `cycle_anchor_clean` 为「日集合」：仅保留出现在该表中的 (id, study_interval, day_in_study)。
- **2.2** 将 1.1–1.3 的日级特征表与 cycle_anchor_clean 做 **右连接**（以 cycle_anchor_clean 为右表），得到「仅完整周期内日」的特征表；缺失的可穿戴/睡眠特征保持 NA（后续模型需支持缺失或插补策略）。

### 阶段 3：Within-individual 标准化（报告 3.2）

- **3.1** 对 HRV、WST 等需要标准化的列：按 (id, study_interval) 分组，按 day_in_study 排序后，对每个 day_in_study = t 计算 \( \mu_t, \sigma_t \)（仅用 t 之前的 K 天；不足 K 天用扩展窗口）；\( z_t = (x_t - \mu_t) / (\sigma_t + \epsilon) \)；\( \sigma_t \) 设下界（如 ε）避免除零。
- **3.2** 输出可覆盖原列（改为 _z 后缀）或新增 _z 列；自报与睡眠可按需做同样标准化或保留原尺度（报告未强制）。

### 阶段 4：监督标签与排卵 0/1

- **4.1** 对每个 (id, study_interval, day_in_study) 在完整周期内：
  - 计算 **days_until_next_menses** = 该周期「下次月经开始日」− 当前 day_in_study（或从 cycle 表取 next_menses_start_day_in_study 再减）；边界处（如周期最后一天）为 0 或 1 视定义而定。
- **4.2** 从 cycle_anchor_clean 的 ovulation_day_method1（及 method2/method3）生成 **ovulation_label**：仅当 day_in_study 等于该周期标注的排卵日时为 1，否则为 0；若某周期无排卵标注，则该周期所有日 ovulation_label = NA（训练时 mask）。

### 阶段 5：自报症状对齐与最终表合并

- **5.1** 从 `hormones_and_selfreport` 取自报症状列（appetite, cramps, fatigue, …），**不取** lh, estrogen, pdg, flow_volume/flow_color 若你希望输入仅可穿戴+睡眠+部分自报则可只取部分；按 (id, study_interval, day_in_study) 与 2.2 结果连接。
- **5.2** 合并所有日级特征 + 周期 ID + day_in_cycle + phase + days_until_next_menses + ovulation_label（+ 可选 method1/2/3 的 0/1 列），得到 **最终训练用日级表**（即 4.3 的「单表」视图）。
- **5.3** 导出为 `daily_features_with_labels.csv`（或拆成 `daily_features.csv` + `daily_labels.csv`，按你实现偏好）。确保表中**无任何激素原始值或激素衍生特征**。

### 阶段 6：数据质量与缺失报告（建议）

- **6.1** 统计每个 (id, study_interval)、每个周期内，HRV/WST/睡眠/自报的缺失比例与缺失模式（如前半周期 vs 后半周期）。
- **6.2** 输出简单报告或表格，便于后续「稀疏采样」评估（如随机 mask 部分日）与论文方法部分描述。
