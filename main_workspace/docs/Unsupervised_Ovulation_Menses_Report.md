# 无监督/规则排卵检测 → 月经预测 实验报告

## 1. 实验目标

- **排卵检测**: 使用纯规则算法/无监督学习，目标 ±2d ≥ 90%
- **月经预测**: 排卵倒计时 + 日历法混合，目标 ±2d ≥ 85%
- **严格约束**: 无数据泄漏、无监督学习标签参与检测

## 2. 数据概况

| 指标 | 数值 |
|------|------|
| 总周期数 | 166 |
| 受试者 | 42 |
| LH标注周期 | 95 |
| 质量周期 (温度偏移≥0.2°C) | 32 (34%) |
| **实际黄体期** | **均值=11.9d, 中位数=12d, 标准差=1.9d** |

**关键发现**: 实际黄体期为 12天，而非常用的14天假设。这个2天的偏差对月经预测影响巨大。

## 3. 实验方法

### 3.1 排卵检测方法 (全部为规则/无监督)

| 方法 | 类型 | 描述 |
|------|------|------|
| 日历法基线 | 规则 | `ov_day = frac × hist_cycle_len` |
| t-test 最优分割 | 规则 | 温度时间序列前后段t检验 + 高斯位置先验 |
| 贝叶斯双相模型 | 规则 | 两段阶梯函数拟合, SSE最小化 + 位置惩罚 |
| Savitzky-Golay 梯度 | 规则 | 平滑后最大正梯度定位 |
| CUSUM | 规则 | 累积和偏差检测 |
| 多信号 t-test | 规则 | 温度+心率+HRV联合t检验 |
| 2-state HMM | 无监督 | 高斯HMM拟合低温/高温两态转换 |
| 集成方法 | 规则 | 多方法加权平均 |
| 置信度混合 | 规则 | 高置信用信号检测，低置信用日历法 |

### 3.2 月经预测流水线 (per-cycle, 无泄漏)

```
对每个周期:
  if 排卵检测置信度 ≥ 阈值:
    predicted_cycle_len = detected_ov_day + luteal_length
  else:
    predicted_cycle_len = 加权历史周期均值 (仅用过去数据)

评估: |predicted_cycle_len - actual_cycle_len| ≤ k天
```

### 3.3 防泄漏设计

- `hist_cycle_len`: 仅来自该受试者**已完成**的过去周期
- `personal_luteal`: 仅从 `past_cycle_len - past_detected_ov` 估算 (不用LH)
- LH标签: **仅用于评估**，从不参与检测算法
- 评估: 按被试级别，保证不使用未来信息

## 4. 核心实验结果

### 4.1 排卵检测准确率

| 数据集 | 最优方法 | MAE | ±1d | ±2d | ±3d | ±5d |
|--------|----------|-----|-----|-----|-----|-----|
| 全部 (95周期) | 双相+t-test集成 | 3.4d | 30% | 51% | **63%** | 85% |
| **质量周期 (32)** | 双相+t-test (σ1.5) | **2.2d** | 34% | **72%** | **84%** | 94% |

**质量周期上排卵检测 ±3d=84.4%，接近目标！** 但全部周期只有63%。

### 4.2 Oracle上限 (完美排卵检测)

| 场景 | 黄体期 | MAE | ±1d | ±2d | ±3d |
|------|--------|-----|-----|-----|-----|
| 全部标注 (95) | 固定12天 | 1.79d | 43% | 67% | **97%** |
| 全部标注 (95) | 固定13天 | 1.55d | 53% | **81%** | **91%** |
| 全部标注 (95) | 个人黄体 | 1.54d | 55% | 77% | **93%** |
| 质量 (32) | 固定13天 | 1.50d | 56% | **84%** | **88%** |
| 质量 (32) | 个人黄体 | 1.37d | 59% | 75% | **94%** |

**Oracle ±3d=93-97%，说明排卵倒计时方法本身非常有效！**

### 4.3 月经预测 (per-cycle)

#### 全部标注周期 (95)

| 方法 | MAE | ±1d | ±2d | ±3d | ±5d |
|------|-----|-----|-----|-----|-----|
| 纯日历法 | 4.16d | 24% | 39% | 50% | 70% |
| **检测+luteal=12** | **3.36d** | **33%** | **51%** | **65%** | 79% |
| 检测+luteal=13 | 3.33d | 35% | 50% | 64% | 81% |
| 置信度路由 (conf≥3, shift≥0.2, luteal=12) | 3.68d | 27% | 45% | 59% | 76% |
| 高置信ov-countdown子集 (31周期) | **2.71d** | 36% | **55%** | **74%** | 87% |

#### 质量周期 (32)

| 方法 | MAE | ±1d | ±2d | ±3d | ±5d |
|------|-----|-----|-----|-----|-----|
| 纯日历法 | 4.32d | 25% | 41% | 44% | 66% |
| **DetQ + luteal=13** | **2.53d** | **41%** | **63%** | **75%** | 84% |
| DetQ + luteal=12 | 2.53d | 47% | 59% | 69% | 84% |

## 5. 关键发现

### 5.1 排卵检测

1. **质量周期上准确率显著**: ±3d=84.4%，MAE=2.2d (接近研究论文水平)
2. **非质量周期是瓶颈**: 66%的周期没有明显温度偏移，检测退化为日历法
3. **多信号(HR+HRV)未显著帮助**: 可能因信号噪声或不同步
4. **集成方法有帮助**: 但改进有限 (~3-5pp)
5. **HMM(无监督)不如规则方法**: 收敛不稳定，准确率低于t-test/biphasic

### 5.2 黄体期

1. **实际均值=12天 (非14天!)**: 使用正确黄体期使Oracle ±3d从84%→97%
2. **个人黄体估计因检测误差传播反而更差**: 固定12-13天优于自适应
3. **13天是 ±2d 最优**: Oracle ±2d=81% (vs luteal=14的76%)

### 5.3 月经预测

1. **排卵倒计时比日历法好**: ±3d: 65% vs 50% (labeled), 75% vs 44% (quality)
2. **高置信子集效果极好**: conf≥5的周期 ov-countdown ±3d=72%, ±5d=90%
3. **混合策略是最优**: 高置信用倒计时，低置信用日历法

### 5.4 与目标的差距

| 目标 | 最优结果 | 差距 | 原因 |
|------|----------|------|------|
| ±2d 排卵 ≥90% | 72% (quality) | **18pp** | SNR不足, 66%周期无明显偏移 |
| ±2d 月经 ≥85% | 63% (quality) | **22pp** | 排卵检测误差 + 黄体变异 |
| ±3d 排卵 ≥90% | **84%** (quality) | **6pp** | 接近! |
| ±3d 月经 ≥85% | 75% (quality) | **10pp** | 主要受检测误差限制 |

## 6. 瓶颈分析

```
目标: ±2d 月经预测 ≥85%

所需条件:
  1. 排卵检测 MAE ≤ 1.5d (当前: 2.2d on quality, 3.4d on all)
  2. 黄体变异 std ≤ 1.5d (当前: 1.9d)
  3. 检测覆盖率 ≥ 80% (当前: 34% quality cycles)

主要瓶颈 (按影响大小排序):
  ① 数据质量: 66%周期无明显BBT偏移 → 检测不可靠
  ② 排卵定位精度: MAE=2.2d (质量周期) → ±2d准确率受限
  ③ 黄体变异性: std=1.9d → 即使完美检测也有 ~15% ±2d误差
```

## 7. 结论

1. **排卵检测在质量周期上已近目标** (±3d=84.4%)，但全周期受数据限制
2. **月经预测在质量周期上显著优于日历法** (±3d: 75% vs 44%，+31pp)
3. **黄体期=12天的发现**是关键校正，提升Oracle性能12pp
4. **规则方法vs监督学习**: 规则方法在质量周期上已达到相当水平，且无过拟合风险
5. **最大瓶颈是数据质量**: 2/3的周期缺乏可检测的温度偏移信号
